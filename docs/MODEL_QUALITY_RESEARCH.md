## Diagnostics

Generated by:

```bash
python -m scripts.diagnose_model_quality --config configs/default.yaml --checkpoint artifacts/models/best_model.pt
```

Findings:

- Training/evaluation windows are annotation-centered MIT-BIH beat windows with fixed length `360`.
- API inference uses detected R-peaks, so API behavior depends on peak-detection quality.
- Test `A` support is low (`139` beats), making A metrics high-variance.
- `L` has non-trivial train and test support, but baseline F1 remains `0.0000`, which points to record-level morphology generalization failure.
- `R` is mostly confused with other morphology classes and remains severe despite threshold tuning.
- Training loss decreases while validation macro F1 plateaus, indicating overfitting and record-level domain shift.

The high fold variance reinforces that MIT-BIH record identity and morphology distribution are major sources of instability.

## Interpretation

A/L/R are failing for different reasons:

- `A`: low held-out support and high confusion with `V`.
- `L`: baseline model does not learn record-generalizable bundle-branch morphology; `resnet1d` can recover it under a limited run.
- `R`: strong record-level failure and unstable architecture behavior.

## Next Research Step

Run uncapped or higher-budget `resnet1d` and a stronger InceptionTime variant with repeated grouped record-level training, then tune thresholds on each validation fold. Promotion should require improved macro F1 plus non-zero and plausible `A`, `L`, and `R` recall on held-out records.
